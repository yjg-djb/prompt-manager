# VIBE：依赖升级（Dependency Upgrade）
> 目标：安全、可回滚地升级依赖（含破坏性变更处理）。

## 输入变量
- 目标依赖/版本范围：{{dependency_and_target_version}}
- 升级原因（安全/功能/兼容）：{{reason}}
- 约束：{{constraints}}

## 工作流程
1. **影响评估**
   - 梳理 breaking changes（迁移指南/变更日志）。
2. **升级策略**
   - 小步：先升级核心依赖，再升级周边；每步都能通过测试。
3. **迁移修改**
   - 按官方迁移建议调整 API、配置、构建脚本。
4. **验证**
   - 单测/集成测试/构建；必要时跑关键路径冒烟。
5. **回滚**
   - 记录回滚方式（锁版本/恢复 lockfile/撤销迁移）。

## 输出格式
- 📦 升级清单：依赖与版本
- 🧩 迁移要点：做了哪些适配
- ✅ 验证：跑了哪些命令/用例
- ⚠️ 风险与回滚：怎么退回
