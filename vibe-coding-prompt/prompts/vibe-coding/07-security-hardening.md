# VIBE：安全加固（Security Hardening）
> 目标：补齐输入校验、鉴权、敏感信息处理与安全默认值。

## 输入变量
- 目标模块/接口：{{target}}
- 威胁模型/关注点：{{threats}}
- 约束（合规/兼容）：{{constraints}}

## 工作流程
1. **资产与边界**
   - 输入来源、信任边界、敏感数据流向。
2. **风险扫描（最常见）**
   - 注入（SQL/命令/模板）、XSS、CSRF、SSRF、路径穿越
   - 权限绕过、IDOR、明文日志、弱随机、错误信息泄露
3. **加固实现**
   - 校验/净化、最小权限、统一鉴权中间件、输出编码、重试与超时。
4. **安全测试**
   - 最少：1 个恶意输入用例 + 1 个权限边界用例。
5. **文档**
   - 安全注意事项、默认配置说明。

## 输出格式
- 🧯 风险点清单：逐条列出
- 🔐 加固改动：对应风险点的改动
- 🧪 安全用例：新增测试与运行方式
- 📌 余留风险：无法一次解决的项与后续建议
