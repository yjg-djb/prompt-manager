# VIBE：补测试（Add Tests）
> 目标：为指定模块建立可靠的测试网（优先覆盖风险最高路径）。

## 输入变量
- 被测对象/模块：{{target}}
- 当前问题/风险：{{risk}}
- 测试类型偏好（单测/集成/端到端）：{{test_type}}
- 运行命令（若已知）：{{test_command}}

## 工作流程
1. **挑选测试策略**
   - 单测：快速、隔离；集成：覆盖真实依赖；E2E：覆盖关键用户路径。
2. **列测试用例**
   - 至少：成功路径、失败路径、边界条件（空/极值/超时/权限）。
3. **实现测试**
   - 复用项目现有测试工具与约定（fixtures/mocks/helpers）。
4. **稳定性**
   - 避免脆弱断言（时间/随机/顺序）；必要时引入假时钟/固定种子。
5. **报告**
   - 输出覆盖了哪些风险点、还缺什么。

## 输出格式
- 🧪 新增测试文件：清单
- 📋 用例列表：用例 -> 断言点
- ▶️ 运行方式：命令 + 注意事项
- 🧯 稳定性说明：如何避免 flaky
