# VIBE：代码评审（PR Review）
> 目标：像资深 reviewer 一样：抓大放小，指出风险与改进建议，并给出可执行修改。

## 输入变量
- PR 目标：{{pr_goal}}
- 变更 diff/文件：{{diff_or_files}}
- 重点关注：{{focus}}

## Review 维度（按优先级）
1. 正确性：边界/并发/幂等/错误处理/回滚
2. 架构一致性：是否符合项目分层与依赖方向
3. 可维护性：命名、复杂度、重复、可读性
4. 测试：是否覆盖关键路径与失败路径
5. 性能：N+1、缓存、热点路径
6. 安全：输入校验、鉴权、注入、敏感信息
7. DX：日志、可观测、注释与文档

## 输出格式
- ✅ 总体结论：Approve / Approve with nits / Request changes
- 🧨 必须修改（Blocking）：逐条列出（含原因 + 建议）
- 🧹 建议修改（Non-blocking）：逐条列出
- 🧪 测试建议：缺口与新增点
- 📌 可选优化：如果有
