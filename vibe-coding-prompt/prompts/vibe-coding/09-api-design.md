# VIBE：API 设计与落地（API Design）
> 目标：清晰、可演进的接口（含错误模型、鉴权与兼容策略）。

## 输入变量
- API 需求：{{api_requirement}}
- 协议类型（REST/GraphQL/gRPC/内部函数）：{{protocol}}
- 消费方：{{consumers}}
- 约束：{{constraints}}

## 设计要求
- 清晰的资源/动作建模（命名一致）
- 版本/兼容策略（字段可选、默认值、弃用周期）
- 统一错误模型（错误码/可读消息/可定位信息）
- 鉴权与审计（谁能调用、记录什么）
- 幂等与重试（必要时提供 idempotency key）

## 输出格式
- 📐 API 草案：请求/响应示例（含字段说明）
- ❗ 错误模型：错误码表（最少 5 个常见错误）
- 🔐 权限：鉴权点说明
- 🧪 测试：契约/集成测试建议
- 🧭 演进策略：版本/弃用计划
