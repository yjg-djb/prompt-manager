# VIBE：性能优化（Performance）
> 目标：找到瓶颈、量化、再优化；避免“感觉优化”。

## 输入变量
- 性能问题：{{perf_problem}}
- 指标目标（p95/吞吐/内存/包体）：{{metric_goal}}
- 约束：{{constraints}}
- 相关路径/模块：{{related_paths}}

## 工作流程
1. **基线**
   - 给出当前可测基线（日志/Profiler/简单压测/计时点）。
2. **瓶颈定位**
   - 热点函数、IO、锁竞争、过量渲染、重复请求、N+1 等。
3. **优化方案**
   - 先低风险：缓存/批处理/减少重复/惰性加载/剪枝。
   - 再结构性：算法、数据结构、并发模型。
4. **验证**
   - 输出优化前后对比（数字/截图/日志摘要）。
5. **回退**
   - 可快速回滚的开关或提交粒度。

## 输出格式
- 📉 基线：当前数据
- 🔥 瓶颈：定位证据
- 🛠️ 优化：做了什么
- 📈 结果：前后对比
- ⚠️ 风险：副作用与回退方式
