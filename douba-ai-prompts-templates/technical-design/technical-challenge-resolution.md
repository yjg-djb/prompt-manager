# 场景：AI技术落地项目技术难点拆解与解决方案设计

## 核心用途

针对AI技术落地项目中的核心技术难点，进行系统化拆解并给出可落地的解决方案。

---

## 提示词模板

### 项目背景

- **项目名称/类型**：[说明项目信息]
  - 示例：Flowise+Ollama企业知识库问答系统
  - 示例：Ollama模型微调优化项目
  - 示例：FastAPI+Ollama高并发接口开发

- **项目阶段**：[说明当前所处阶段]
  - 需求分析阶段 / 开发阶段 / 测试阶段 / 部署阶段

- **核心目标**：[说明项目要达成的目标]
  - 示例：实现离线环境下的精准问答
  - 示例：提升模型特定领域问答准确率
  - 示例：支持100人并发的文本生成接口

### 技术难点详情

1. **难点描述**：[详细描述遇到的技术问题]
   - 示例：离线环境下Ollama模型无法下载部署
   - 示例：大文件PDF解析速度慢（50MB文件需30分钟）
   - 示例：高并发下接口响应超时（并发>50时超时率>30%）
   - 示例：模型生成答案与知识库内容不一致

2. **影响范围**：[说明该问题造成的影响]
   - 示例：导致项目无法离线部署
   - 示例：影响用户体验，无法满足批量文档处理需求
   - 示例：无法支撑预期用户规模使用
   - 示例：降低系统可信度

3. **已尝试方案及效果**：[说明已做的尝试]
   - 示例：尝试压缩PDF后解析，效果不明显
   - 示例：增加服务器内存，并发能力无显著提升

4. **约束条件**：[说明解决方案的限制]
   - 示例：无GPU资源、服务器内存≤16GB
   - 示例：不能修改开源组件核心代码
   - 示例：需兼容Windows系统

### 期望输出要求

1. **难点核心原因分析**
   - 从技术原理、资源限制、环境约束等角度拆解

2. **难点拆解**
   - 将核心难点拆解为可解决的子问题
   - 按优先级排序

3. **针对性解决方案**
   - 每个子问题对应具体解决方案
   - 含技术选型、操作步骤、配置参数

4. **方案可行性验证方法**
   - 如何确认方案有效
   - 含测试用例、评估指标

5. **实施计划**
   - 方案实施的步骤
   - 所需资源、时间预估

6. **备选方案**（主方案不可行时的替代方案）

7. **风险评估**（方案实施过程中可能遇到的风险及应对措施）

### 输出格式约束

- 用章节式结构（一、原因分析 → 二、难点拆解 → ...）
- 子问题用有序列表
- 解决方案分点清晰
- 测试用例用表格呈现
- 风险评估用表格说明

## 使用示例

```
我的AI项目遇到了技术瓶颈，需要帮助分析解决。

【项目背景】
- 项目名称/类型：FastAPI+Ollama企业文档问答API服务
- 项目阶段：测试阶段
- 核心目标：为企业内部200人提供文档问答服务，支持50人并发

【技术难点详情】
1. 难点描述：
   - 并发测试时，超过20个并发请求后响应时间急剧上升（从2s上升到15s+）
   - 并发超过30时开始出现大量超时错误
   - CPU占用100%，但GPU利用率仅10%

2. 影响范围：
   - 无法满足50人并发的性能指标
   - 用户体验差，可能导致项目无法上线

3. 已尝试方案及效果：
   - 增加uvicorn worker数量（4→8），略有改善但不明显
   - 使用异步调用Ollama API，效果不明显
   - 升级服务器配置（8核→16核），CPU占用下降但并发能力未提升

4. 约束条件：
   - 服务器配置：16核CPU、64GB内存、RTX 3080（10GB显存）
   - 必须使用Ollama本地部署，不能使用云API
   - 模型为qwen:7b，不能更换更小的模型

请帮我分析问题原因并提供系统化的解决方案。
```
