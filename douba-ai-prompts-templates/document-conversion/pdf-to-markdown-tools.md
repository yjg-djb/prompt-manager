# 场景：批量PDF转Markdown工具选型与使用指南

## 核心用途

根据转换需求选择适配的工具，获取具体操作步骤及格式优化建议。

---

## 提示词模板

### 转换需求

- **文档规模**：[说明PDF数量和大小]
  - 示例：100份PDF文档，单文档页数5-50页
  - 示例：总容量10GB，含扫描版/可编辑版PDF

- **格式要求**：[说明转换后的格式期望]
  - 示例：保留图片、表格结构、公式
  - 示例：去除水印、页眉页脚
  - 示例：保持段落层级

- **转换效率**：[说明效率要求]
  - 需批量一次性转换 / 允许单份转换，优先保证格式准确性

- **运行环境**：[说明系统环境]
  - Windows10/macOS/Linux
  - 是否支持离线运行
  - 是否需要GUI界面

- **工具偏好**：[说明工具类型偏好]
  - 开源工具 / 免费软件 / 付费工具 / Python脚本

### 额外需求

- **是否需要批量命名**：[说明命名规则]
  - 示例：按原PDF文件名命名MD文件
  - 示例：按"文档类型+日期"命名

- **是否需要后处理**：[说明后处理需求]
  - 示例：统一MD格式（标题层级、列表样式）
  - 示例：合并多个MD文件

- **特殊文档类型**：[说明特殊内容]
  - 示例：含大量公式的学术PDF
  - 示例：含流程图的技术文档
  - 示例：扫描版图片PDF

### 期望输出要求

1. **Top3适配工具推荐**（按适配度排序，含工具名称、类型、开源/收费状态）
2. **每款工具的详细信息**：
   - 安装步骤
   - 批量转换操作流程（含命令/界面操作）
   - 优缺点
   - 适配场景
3. **优先推荐工具的详细使用指南**（含批量转换配置、格式优化设置）
4. **常见问题解决方案**（如格式错乱、图片丢失、公式无法识别）
5. **后处理优化建议**（如MD格式统一工具、手动调整技巧）

### 输出格式约束

- 工具推荐用表格呈现
- 安装及操作步骤用有序列表
- 命令用代码块标记
- 优缺点分点列出
- 注意事项用高亮块呈现

---

## 使用示例

```
我需要将大量技术文档PDF转换为Markdown格式。

【转换需求】
- 文档规模：200份技术文档PDF，单文档10-100页，总量约15GB
- 格式要求：保留代码块格式、表格结构、图片；去除页眉页脚
- 转换效率：需要批量转换，可以接受部分格式需手动调整
- 运行环境：Windows11，需支持离线运行，命令行即可
- 工具偏好：优先开源工具或Python脚本

【额外需求】
- 是否需要批量命名：按原文件名命名
- 是否需要后处理：需要统一标题层级（H1-H4）
- 特殊文档类型：部分文档含代码截图和流程图

请推荐适合的工具并提供详细使用指南。
```
