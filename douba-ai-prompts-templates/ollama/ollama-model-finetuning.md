# 场景：Ollama模型微调需求梳理与指令生成

## 核心用途

明确模型微调的需求边界，生成适配Ollama微调流程的精准指令。

---

## 提示词模板

### 微调基础信息

- **基础模型**：[指定待微调的模型]
  - 示例：`llama3:8b`、`mistral:7b`、本地已下载模型/需先下载

- **微调目标**：[描述微调期望达成的效果]
  - 示例：优化特定领域（医疗/法律/技术）问答准确性
  - 示例：让模型适配企业内部术语
  - 示例：提升文本生成的风格一致性（如正式报告风格）

- **部署环境**：[说明硬件环境]
  - 示例：Windows10（CPU/GPU）、Linux Ubuntu 22.04（GPU: NVIDIA RTX 3090）
  - 显存大小：24GB

### 数据信息

- **训练数据类型**：[说明数据格式]
  - 示例：JSON格式问答对、TXT文本corpus、CSV表格数据

- **数据规模**：[说明数据量]
  - 示例：1000条问答对、50万字文本

- **数据特点**：[描述数据特殊性]
  - 示例：含企业内部产品术语、涉及专业技术参数、需过滤敏感信息

### 微调约束

- **性能要求**：[说明性能指标]
  - 示例：微调后模型响应时间<2s、生成文本准确率≥85%

- **资源限制**：[说明资源约束]
  - 示例：微调耗时≤24h、显存占用≤20GB

- **输出要求**：[说明输出期望]
  - 示例：微调后的模型保留原基础模型的通用能力、生成内容符合企业品牌话术规范

### 期望输出要求

1. **微调可行性分析**（基于基础模型、数据、环境的适配性）
2. **数据预处理步骤**（格式转换、清洗、标注建议）
3. **Ollama微调命令及参数配置**（含具体命令、参数说明、推荐值）
4. **微调过程监控指标及异常处理**
5. **微调后模型测试与评估方法**（测试用例、评估维度）

### 输出格式约束

- 用章节式结构（一、可行性分析 → 二、数据预处理 → ...）
- 命令用代码块标记
- 参数用表格说明
- 注意事项用高亮块呈现

---

## 使用示例

```
我需要对Ollama模型进行微调以适配企业场景。

【微调基础信息】
- 基础模型：llama3:8b（已下载）
- 微调目标：优化模型在法律领域的问答能力，准确理解法律术语和条款
- 部署环境：Linux Ubuntu 22.04、NVIDIA RTX 4090、48GB显存

【数据信息】
- 训练数据类型：JSON格式问答对
- 数据规模：5000条法律问答对
- 数据特点：包含民法、刑法、合同法等领域的专业术语

【微调约束】
- 性能要求：法律问答准确率≥90%、响应时间<3s
- 资源限制：微调时间≤48h
- 输出要求：保留通用问答能力，法律回答需严谨专业

请提供完整的微调方案，包含可行性分析、数据预处理、微调命令和评估方法。
```
