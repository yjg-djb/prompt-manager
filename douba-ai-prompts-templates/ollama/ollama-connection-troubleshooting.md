# 场景：Ollama服务启动/连接异常排查

## 核心用途

快速定位Ollama服务无法启动、客户端连接失败等问题，获取针对性解决方案。

---

## 提示词模板

### 问题场景

- **操作目标**：[描述你要完成的操作]
  - 示例：启动Ollama服务、通过Python脚本连接Ollama服务调用模型、Flowise连接Ollama服务

- **报错现象**：[描述观察到的问题]
  - 示例：服务启动后秒退、调用时提示`connection refused`、模型下载中断

- **完整错误信息**：[粘贴命令行输出、弹窗提示或日志内容]

### 环境信息

- **操作系统**：[填写系统版本]
  - 示例：Windows10 22H2、macOS Sonoma 14.0、Linux Ubuntu 22.04

- **Ollama版本**：[填写版本号]
  - 示例：v0.1.28、v0.1.30、v0.3.0

- **网络环境**：[说明网络情况]
  - 本地局域网/公网、是否需代理

- **硬件信息**：[说明硬件配置]
  - CPU型号、内存大小、是否有GPU及型号

- **已尝试操作**：[列出已进行的排查措施]
  - 示例：重启电脑、重新安装Ollama、检查端口占用

### 期望输出要求

1. **核心错误原因分析**（1-2句明确结论）
2. **分步排查步骤**（从基础环境到具体配置，每步含操作命令/方法）
3. **针对性解决方案**（按排查结果给出具体操作，如修改配置、释放端口、调整网络）
4. **服务验证方法**（确认服务正常运行的测试步骤）
5. **长期稳定运行建议**（如服务自启动配置、资源占用优化）

### 输出格式约束

- 按"原因分析 → 排查步骤 → 解决方案 → 验证方法 → 稳定建议"的顺序呈现
- 操作命令用代码块标记
- 步骤用有序列表
- **关键检查点**用加粗标注

```
【问题场景】
- 操作目标：启动Ollama服务并通过Python调用模型
- 报错现象：运行`ollama serve`后服务秒退，无任何输出
- 完整错误信息：（无错误提示，直接退出）

- Ollama版本：v0.1.30
- 网络环境：本地局域网，无代理
- 硬件信息：Intel i7-12700、32GB内存、NVIDIA RTX 3070
- 已尝试操作：重启电脑、以管理员身份运行、检查11434端口未被占用

请帮我分析问题原因并提供解决方案。
```
