# 场景：Flowise节点配置/连接问题排查

## 核心用途

快速定位Flowise各类节点（Ollama、向量数据库、API等）配置或连接失败问题，获取分步解决方案。

---

## 提示词模板

### 技术场景信息

- **Flowise版本**：[填写具体版本，如 `v1.6.0`]
- **涉及节点类型**：[如 Ollama节点、Chroma向量数据库节点、自定义API节点]
- **运行环境**：[如 Windows10/macOS/Linux、本地部署/服务器部署]

### 问题详情

1. **操作步骤**：[按时间顺序填写操作]
   - 示例：1. 新建Ollama节点；2. 输入模型名称`llama3:8b`；3. 填写Ollama服务地址`localhost:11434`；4. 点击测试连接报错

2. **完整错误信息**：[粘贴报错弹窗文字或日志内容]
   - 示例：`Error: Failed to connect to Ollama. Could not resolve host 'localhost:11434'`

3. **已尝试措施**：[如已进行的排查操作]
   - 示例：1. 检查Ollama服务已启动；2. 测试11434端口是否占用

### 期望输出要求

1. **错误原因核心分析**（1-2句明确结论）
2. **分步解决方案**（每步操作需具体，含文件路径、配置参数等）
3. **验证步骤**（如何确认问题已解决）
4. **预防建议**（避免后续重复出现的配置规范）

### 输出格式约束

- 严格按"一、原因分析 → 二、解决方案 → 三、验证步骤 → 四、预防建议"分章节
- 代码片段、文件路径用 ``` 标记
- 步骤用有序列表呈现

---

## 使用示例

```
我在使用Flowise v1.6.0时遇到了节点连接问题。

【技术场景信息】
- Flowise版本：v1.6.0
- 涉及节点类型：Ollama节点
- 运行环境：Windows10本地部署

【问题详情】
1. 操作步骤：新建Ollama节点 → 输入模型名称llama3:8b → 填写服务地址localhost:11434 → 点击测试连接
2. 完整错误信息：Error: connect ECONNREFUSED 127.0.0.1:11434
3. 已尝试措施：重启Ollama服务、检查防火墙设置

请按照"原因分析→解决方案→验证步骤→预防建议"的格式帮我排查问题。
```
