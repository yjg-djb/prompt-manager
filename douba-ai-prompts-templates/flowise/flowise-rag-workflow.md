# 场景：基于Flowise的RAG知识库问答流程搭建

## 核心用途

明确RAG流程搭建的需求输入项，生成分步搭建指南及参数配置建议。

---

## 提示词模板

### RAG项目需求

- **应用场景**：[描述具体业务场景]
  - 示例：企业内部产品文档问答、个人知识库检索、客户服务FAQ问答

- **知识库数据源**：[说明数据类型和规模]
  - 示例：PDF文档（100份）、Word文档、TXT文本

- **核心要求**：[列出关键性能和功能要求]
  - 示例：响应时间<1s、支持模糊检索、准确引用文档来源、可离线运行

- **技术栈约束**：[指定技术选型]
  - 示例：向量数据库用Chroma（本地）、大模型用Ollama-llama3:8b、Flowise v1.6.0

### 基础信息

- **部署环境**：[操作系统、硬件配置]
  - 示例：Windows10/macOS/Linux、CPU/GPU支持情况

- **文档处理规模**：[说明数据量]
  - 示例：单文档最大50MB，总文档量10GB

- **用户交互方式**：[说明使用方式]
  - 示例：Web界面提问、API调用

### 期望输出要求

1. **Flowise RAG流程架构图说明**（用文字描述核心节点及流转逻辑）
2. **分步搭建步骤**：
   - 文档加载 → 文本分割 → 向量生成 → 向量存储 → 检索问答 → 结果输出
3. **各节点核心参数配置建议**（如文本分割长度、向量模型选择、检索相似度阈值）
4. **流程测试方法及优化方向**
5. **部署上线注意事项**（性能优化、数据备份）

### 输出格式约束

- 步骤按流程阶段分章节
- 参数配置用表格呈现
- **关键操作节点**用加粗标注
- 优化方向用项目符号列出

---

## 使用示例

```
我需要搭建一个企业内部的RAG知识库问答系统。

【RAG项目需求】
- 应用场景：企业内部技术文档问答，供研发团队使用
- 知识库数据源：PDF技术手册200份、Word规范文档50份，总量约5GB
- 核心要求：响应时间<2s、支持模糊语义检索、返回答案需引用文档来源
- 技术栈约束：Flowise v1.8.0、Ollama-qwen:7b、Chroma向量数据库

【基础信息】
- 部署环境：Linux Ubuntu 22.04、32GB内存、NVIDIA RTX 3080 GPU
- 文档处理规模：单文档最大100MB
- 用户交互方式：内部Web界面 + API接口

请提供完整的RAG流程搭建指南，包含架构说明、配置步骤和参数建议。
```
